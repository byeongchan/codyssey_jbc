## 1. LLM 모델 비교 및 선정 보고서

| 평가 축 | Gemini 3 Flash | GPT-5.4 mini | Claude Haiku 4.5 |
| --- | --- | --- | --- |
| 응답 정확성 | 4 | 4 | 5 |
| 응답 속도 | 3 | 5 | 5 |
| 비용 효율성 | 5 | 5 | 3 |
| 문서 작성 품질 | 4 | 3 | 5 |
| 총점 | 16 | 17 | 18 |

- __최종 선정 결론__ : 응답속도는 gemini 3 flash가 가장 빨라야 하나 네이토 상에서 가장 느린 답변을 하여 낮은 점수를 줌.
Claude Haiku 4.5 가 세 모델 중 가장 비싸지만 문서 작성 품질과 응답 정확성면에서 성능이 더 우수하였음. Gemini 3 Flash와 GPT-5.4 mini는 입력되지 않은 정보에 대해 질문하지 않고 예시로 작성하였으나 Claude Haiku 4.5는 추가 질문을 수행하는 비율이 높았으며, 사실을 추측하여 작성하는 환각 발생 빈도가 상대적으로 낮음. 메일 작성 업무에서는 사용자가 제공하지 않은 일정, 금액, 담당자 정보를 임의 생성하지 않는 것이 중요하여 Claude Haiku 4.5를 선정함.

[__비용 제약 환경에서의 프롬프트 및 프로세스 개선 전략__]

- 입력 템플릿 표준화
    - 사용자가 목적, 수신 대상, 핵심 내용, 원하는 톤, 필수 포함 내용, 금지 표현 등을 구조화된 형태로 입력하도록 함. 이를 통해 모델이 부족한 정보를 추론하거나 환각을 발생시키는 가능성을 줄임
- 단계적 정보 수집 프로세스 적용
    - 필수 정보가 누락된 경우 즉시 메일을 생성하지 않고 추가 질문을 수행하도록 함. 특히 일정, 금액, 정책, 담당자 정보와 같은 사실 정보는 반드시 사용자 입력을 통해 확보하도록 설계.
- Few-shot 예시 활용
    - 좋은 입력과 출력 예시를 시스템 프롬프트에 포함하여 상대적으로 성능이 낮은 모델에서도 일관된 출력 형식을 유지하도록 함.
- 출력 형식 고정
    - 제목, 본문, 마무리 문구와 같이 출력 구조를 고정하여 모델의 자유도를 줄이고 결과 품질의 편차를 감소시킴.

[__업무 정책 변경 시 환각 방지를 위한 운영 전략__]

- 정책 정보의 외부 관리
    - 변경 가능성이 높은 정보는 시스템 프롬프트에 직접 작성하지 않고, 별도의 정책 문서 또는 사내 지식 저장소에서 관리한다.
- 정책 버전 관리
    - 업무 정책 변경 시 버전 번호와 적용 일자를 관리하여 AI가 최신 정책을 참조하도록 한다.
- 확인 필요 규칙 적용
    - 사용자가 제공하지 않았거나 최신 여부를 확인할 수 없는 정책, 일정, 수치 정보에 대해서는 임의 생성하지 않고 "확인 필요"로 표시하도록 한다.
- 정기적인 프롬프트 및 Few-shot 업데이트
    - 조직 정책 변경 시 시스템 프롬프트와 Few-shot 예시를 함께 수정하여 최신 업무 환경을 반영한다.

[__다중 턴 대화에서의 문맥 단절 방지 전략__]
- 슬롯 기반 정보 관리
    - 메일 작성에 필요한 정보를 슬롯 형태로 관리한다. 새로운 입력이 들어오면 기존 슬롯 값을 유지하고 변경된 항목만 업데이트한다.
- 현재 상태 요약 제공
    - 각 대화 턴마다 현재까지 수집된 정보를 요약하여 사용자와 공유한다.
- 필수 정보와 선택 정보 구분
    - 필수 정보가 모두 수집되면 추가적인 선택 정보가 없더라도 메일 초안을 생성하도록 함.

- 당신은 기업용 메일 작성 전문가이다. 사용자의 입력 정보를 바탕으로 비즈니스 메일 초안을 작성한다.

- 제목과 본문을 구분한다.
- 사용자가 요청한 톤을 유지한다.
- 일정, 수치, 정책 정보는 확인되지 않은 경우 "확인 필요"라고 표시한다.
- 최종 출력은 간결하게 작성한다.
